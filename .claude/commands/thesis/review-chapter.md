# 챕터 검토 (HITL-5~8)

## 역할

이 커맨드는 **챕터별 HITL 체크포인트**로, 각 장 작성 완료 후 사용자가 검토하고 승인하는 단계입니다.

## 사용법

```
/thesis:review-chapter 1    # 1장 서론 검토 (HITL-5)
/thesis:review-chapter 2    # 2장 이론적 배경 검토 (HITL-6)
/thesis:review-chapter 3    # 3장 연구방법론 검토 (HITL-7)
/thesis:review-chapter 4    # 4장 분석결과 검토 (HITL-8a)
/thesis:review-chapter 5    # 5장 결론 검토 (HITL-8b)
```

## HITL 매핑

| 챕터 | HITL | 설명 |
|------|------|------|
| 1장 | HITL-5 | 서론 검토 |
| 2장 | HITL-6 | 이론적 배경 검토 |
| 3장 | HITL-7 | 연구방법론 검토 |
| 4-5장 | HITL-8 | 분석결과 및 결론 검토 |

## 프로세스

### Step 1: 챕터 파일 확인

`$1` 인자로 전달된 챕터 번호 확인:
- `thesis-output/chapters/chapter-{N}.md` 존재 확인
- 작성 상태 확인

### Step 2: 품질 지표 제시

#### 2.1 SRCS 점수
| 축 | 점수 | 상태 |
|----|------|------|
| CS (출처) | XX | ✅/⚠️ |
| GS (근거) | XX | ✅/⚠️ |
| US (불확실성) | XX | ✅/⚠️ |
| VS (검증가능성) | XX | ✅/⚠️ |
| **종합** | XX | |

#### 2.2 표절 검사 결과
- 유사도: XX%
- 문제 구간: [있음/없음]

#### 2.3 일관성 검사
- 이전 챕터와의 일관성
- 용어 일관성
- 논리적 흐름

### Step 3: 챕터 내용 요약

#### 3.1 주요 내용
- 섹션별 핵심 내용 요약

#### 3.2 핵심 주장
- 챕터의 핵심 클레임 목록
- 각 클레임의 근거

#### 3.3 참고문헌
- 인용 문헌 수
- 주요 인용 문헌

### Step 4: 사용자 검토 요청

1. **내용 완전성**
   - 필수 내용 포함 여부
   - 추가 필요 내용

2. **논리적 흐름**
   - 섹션 간 연결성
   - 논증 구조

3. **학술적 표현**
   - 문체 적절성
   - 전문용어 사용

### Step 5: 결정 사항

```
1. ✅ 승인: 다음 챕터 진행
2. 📝 수정: 특정 부분 수정 요청
3. 🔄 재작성: 전체 재작성 필요
4. ➕ 내용 추가: 특정 내용 보강
```

### Step 6: 승인 처리

승인 시:
1. 챕터 완료 상태 기록
2. 다음 챕터 작성 시작 (해당 시)
3. HITL 체크포인트 완료 기록

## 출력 형식

```markdown

[섹션별 요약]

## 4. 핵심 클레임
| ID | 내용 | 근거 | SRCS |
|----|------|------|------|

## 5. 수정 제안
- [수정 사항 1]
- [수정 사항 2]

## 6. 승인 결정
[ ] 승인
[ ] 수정 후 재검토
[ ] 재작성
```
